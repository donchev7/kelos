# Moontide AI — MVP Business Intent & Decisions

**Date:** 2026-02-17
**Status:** Decisions locked, implementation plan pending

---

## Business Intent

Build the first E2E functional MVP of Moontide AI — an AI agent platform where cloud-deployed agents perform real work (code implementation, PR review, digests, task execution) triggered by GitHub and Slack events, with a verification layer so users review agent output before it ships.

**Target:** Dogfood-ready. Shan + CTO use it daily at their own company doing real work.

**Timeline:** 4-6 weeks.

---

## Core Product Loop (MVP)

```
User connects GitHub + Slack
  → Events flow in (PR opened, Slack message, activity)
  → Agent picks up work in cloud sandbox
  → Agent clones repo, writes code, runs tests, creates PR (or reviews, summarizes, executes)
  → User gets notified with progress updates
  → User reviews agent output in Moontide UI
  → Approves / rejects / modifies
```

**Observer (passive watching/pattern detection) is deferred.** Agents are triggered by direct events, not by "observing" passively.

## Architecture Decisions

### Agent Sandbox Runtime

| Phase | Technology | Isolation | Why |
|---|---|---|---|
| **MVP** | E2B | Firecracker microVM | Ship fast. TypeScript SDK. Same isolation level as target. Zero infra ops. |
| **Growth** | AKS + Kata Containers | Firecracker microVM | Full control. Azure enterprise story. No session limits. Lower per-sandbox cost at scale. |

**Migration cost:** ~2-3 weeks of focused work. Build a `SandboxProvider` abstraction from day 1 so the swap is one implementation change.

### Inner Agent (what runs inside the sandbox)

**Approach:** Wrap an existing MIT-licensed agent CLI — don't build a custom agent loop.

**Candidates to evaluate:**
- **OpenCode** (Go, MIT) — battle-tested at Ramp, powers 30% of their PRs. Headless server mode. 70K+ stars.
- **Pi** (TypeScript, MIT) — minimal 4-tool agent (read, write, edit, bash). Powers OpenClaw (180K+ stars). Proves frontier models don't need complex tooling.

**Decision:** Try both during development, pick the better performer.

### LLM Abstraction

**Vercel AI SDK** (TypeScript, Apache 2.0). Supports all major providers (Claude, GPT, Gemini). Streaming, tool use, structured output built in. Fits the NestJS/TypeScript stack.

### Integrations (MVP)

**GitHub + Slack**, both via OAuth apps.

- GitHub: webhooks for PRs, commits, issues. OAuth for API access (post reviews, create PRs).
- Slack: events API for messages/commands. OAuth for posting messages/digests.

### Auth

**Keep Better Auth** (already working E2E). Extend with org support via additional database tables.

### Tenant Model

**Org shell, single user.** Create an `organizations` entity on signup (name, slug) but only one user per org for now. Future-proofs the data model for team invites without building invite flows.

### Onboarding Flow

```
Sign up (email/password)
  → Create organization (name, slug)
  → Connect GitHub (OAuth)
  → Connect Slack (OAuth)
  → Create first agent (wizard)
  → Done — agent is deployed and listening
```

### Billing

**Stripe from day 1.** Set up subscription infrastructure and usage metering. Actual pricing tiers TBD — launch with invite-only free access. Track:
- Agent sandbox hours
- LLM token consumption
- API calls per integration

### Secrets / Credentials

**AES-256 encrypted in PostgreSQL.** Per-org credentials table for GitHub tokens, Slack tokens, LLM API keys. Simple, works for MVP. Upgrade to a vault (GCP Secret Manager or HashiCorp Vault) when enterprise customers require it.

### Real-Time Communication

**WebSocket** from sandbox → BFF → browser for agent progress updates. Not full streaming of every file change (like Cursor/Devin), but periodic status updates: "cloning repo", "running tests", "creating PR", etc.

### Verification Flow

**Notification + review.** Agent completes work → user gets notified (in-app + Slack) → clicks through to review the output (PR, code changes, digest) with Moontide's context overlay → approves or rejects.

Not a full verification queue for MVP — lighter and faster to build.

### Frontend Strategy

**Keep all 45 existing pages.** Add a demo/real toggle — when demo mode is off, mock data is hidden and only real functional pages are active. No need to delete the demo UI; it's still useful for showing the vision.

### Domain / Hosting

**Staging URL for now.** Don't worry about production domain until closer to launch.

## Agent Capabilities (MVP Scope)

All 4 are stretch goals. If scope pressure forces a cut, prioritize top-to-bottom:

### 1. Code Implementation Agent
**Trigger:** Slack message ("implement feature X in repo Y")
**Action:** Clone repo → plan approach → write code → run tests → create PR
**Output:** GitHub PR with description, linked to Slack thread

### 2. PR Code Review Agent
**Trigger:** GitHub webhook (PR opened)
**Action:** Clone repo → analyze diff → LLM reviews code → generate review comments with confidence
**Output:** GitHub PR review comments, notification in Moontide UI

### 3. Event Digest / Standup Agent
**Trigger:** Scheduled (daily) or on-demand via Slack
**Action:** Aggregate GitHub + Slack activity for the day → LLM summarizes
**Output:** Slack message with daily standup digest

### 4. Slack → Action Bridge
**Trigger:** Slack command/message ("create issue for X", "deploy staging")
**Action:** Parse intent → execute via GitHub/Slack API
**Output:** Confirmation in Slack thread

## What's Deferred (Not MVP)

| Feature | Why Deferred |
|---|---|
| **Observer** (passive activity watching, pattern detection, suggestions) | Core differentiator but complex. Agents are event-triggered for MVP. |
| **Multi-user orgs** (invites, RBAC, roles) | Org shell exists for data model. Invite flows come with first team customer. |
| **Agent config editing** (change model, autonomy, permissions live) | Read-only config in MVP. Wizard creates, no edit. |
| **Full verification queue** (confidence scores, diffs, multi-agent) | Notification + review is sufficient for dogfood. Queue comes with scale. |
| **Full real-time streaming** (file-by-file, terminal output) | Progress updates only. Full streaming is a UX project unto itself. |
| **Production domain + CDN** | Staging URL works for dogfood. |
| **Custom agent logic** (user-defined workflows) | Platform-defined agents only for MVP. |
| **Multi-model per agent** (GPT for code, Claude for review) | Single model per agent via Vercel AI SDK. Abstraction supports swap later. |
